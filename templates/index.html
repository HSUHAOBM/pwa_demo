這不是純靜態頁

按鈕會真的呼叫 Flask API，證明這個 PWA 仍然是有後端能力的 Web App。

PWA 核心元件還在

Manifest 提供安裝資訊，service worker 負責快取與離線 fallback。

適合先做概念驗證

先確認可安裝、可快取、可互動，再決定要不要把既有專案完整 PWA 化。

現在可以測什麼

  • 點「測試 Flask API」，確認前端能收到 Flask JSON 回應。
  • Chrome DevTools 的 Application 面板能看到 Manifest。
  • Service Worker 成功註冊並快取首頁與靜態資源。
  • 手機瀏覽器可顯示安裝提示，或出現加入主畫面的選項。
